## Project Overview

This pipeline performs systematic genome annotation for ~8,000 fungal genomes from NCBI, producing a common orthology dataset for the 1000 Fungal Genomes (1KFG) project. The pipeline runs on a SLURM HPC cluster (UCR HPCC) using Nextflow to orchestrate funannotate (gene prediction) and antiSMASH (secondary metabolite cluster detection).

All Nextflow workflows are launched from the project root (`/bigdata/stajichlab/shared/projects/1KFG/common_annotate/`), not from inside `pipeline/`.

## Running the Pipeline

**Submit the funannotate prediction pipeline:**
```bash
sbatch pipeline/02_predict_nf.sh
```

**Submit the antiSMASH pipeline (runs after prediction):**
```bash
sbatch pipeline/06_antismash_nf.sh
```

Both scripts run Nextflow with `-resume`, so interrupted runs can be restarted safely.

**Test with a small subset:**
```bash
nextflow run pipeline/nextflow/funannotate.nf -profile slurm --n_test 5 -resume
```

**Run locally (single machine, no SLURM):**
```bash
nextflow run pipeline/nextflow/funannotate.nf -profile local -resume
```

## Key Parameters

Both `.nf` workflows read defaults from `nextflow/nextflow.config` and accept CLI overrides:

| Parameter | Default | Description |
|---|---|---|
| `--samples` | `samples.csv` | Input manifest |
| `--target` | `annotate/` | Output directory |
| `--source` | `/bigdata/stajichlab/shared/projects/1KFG/2023/NCBI_fungi/source/NCBI_ASM` | Gzipped genome FNA files |
| `--funannotate_db` | `/bigdata/stajichlab/shared/lib/funannotate_db` | funannotate database |
| `--augustus_config` | `lib/augustus/3.5/config` | AUGUSTUS config directory |
| `--min_contig_len` | 2000 | Minimum contig length for `clean_genome_fa.py` |
| `--debug` | false | Verbose channel/script logging |
| `--n_test` | 0 (all) | Limit to first N samples |

## Samples Manifest (`samples.csv`)

~8,061 rows with header: `ASMID,SPECIES,STRAIN,BIOPROJECT,NCBI_TAXONID,BUSCO_LINEAGE,PHYLUM,SUBPHYLUM,CLASS,SUBCLASS,ORDER,FAMILY,GENUS,SPECIES,LOCUSTAG`

- **ASMID**: NCBI assembly ID + assembly name (e.g., `GCF_010015735.1_Aaoar1`); used to locate the genome at `${source}/${ASMID}/${ASMID}_genomic.fna.gz`
- **LOCUSTAG**: MD5-derived 8-character prefix (`F` + last 7 hex chars of MD5 of ASMID), generated by `scripts/create_samples_file.py`
- **BUSCO_LINEAGE**: `dikarya`, `mucoromycota`, or `fungi` — assigned by taxonomy in `create_samples_file.py`

**Regenerating samples.csv** from `ncbi_accessions.csv` + `ncbi_accessions_taxonomy.csv`:
```bash
python scripts/create_samples_file.py
```

## Architecture

```
project root/
├── samples.csv                  # manifest of all ~8k fungal assemblies
├── ncbi_accessions.csv          # raw NCBI assembly metadata
├── ncbi_accessions_taxonomy.csv # taxonomy + BUSCO lineage assignments
├── annotate/                    # output: one dir per species
│   └── <Species_name>/
│       ├── predict_results/     # funannotate output (*.gbk = completion marker)
│       ├── predict_misc/        # intermediate files (cleaned up after success)
│       └── antismash_local/     # antiSMASH output (*.json.gz = completion marker)
├── pipeline/
│   ├── 02_predict_nf.sh         # SLURM submission wrapper for funannotate
│   ├── 06_antismash_nf.sh       # SLURM submission wrapper for antiSMASH
│   └── nextflow/
│       ├── funannotate.nf       # gene prediction workflow (DSL2)
│       ├── antismash.nf         # BGC detection workflow (DSL2)
│       └── nextflow.config      # SLURM/local profiles + per-process resource specs
└── scripts/
    ├── clean_genome_fa.py       # filters short/low-complexity contigs (stdin→stdout)
    ├── create_samples_file.py   # generates samples.csv from NCBI metadata
    ├── cleanup_predict_misc.sh  # parallel cleanup of large intermediate dirs post-run
    └── find_unfinished.sh       # identifies assemblies missing predict_results gbk
```

### Skip Logic

Both workflows skip samples that already have output, making them safe to re-run:
- `funannotate.nf`: skips if `annotate/<Species>/predict_results/*.gbk` exists
- `antismash.nf`: skips if `annotate/<Species>/antismash_local/*.json` or `*.json.gz` exists

### Genome Cleaning

`scripts/clean_genome_fa.py` filters contigs by minimum length and requires ≥4 distinct nucleotide characters (removes low-complexity/contamination). Called with `--len` flag. Note: the `.nf` workflow uses `--len` but the script argument is `--length`; verify consistency when modifying.

## Cluster Configuration

- **Cluster**: UCR HPCC, SLURM scheduler, `epyc` queue
- **funannotate**: 16 CPUs, 128 GB RAM, 72h walltime (per `nextflow.config` slurm profile)
- **antiSMASH**: 8 CPUs, 16 GB RAM, 4h walltime
- Queue limit: 200 concurrent jobs, submit rate 20/min
- Nextflow logs: `logs/funannotate-report.html`, `logs/funannotate-trace.txt`
- Work directories: `work.funannotate/`, `work.antismash/` (separate per pipeline)

## Known Exclusions

Several species fail prediction due to being Microsporidia, too-small genome deposits, or contamination — see `NOTES.md` for the full exclusion list. Notable long-runtime cases: *E. muscae* and *A. psidii*.
